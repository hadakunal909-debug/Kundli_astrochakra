# Deploying Astrochakra to Vercel

This is an npm-workspace monorepo:
- root = the `@prisri/jyotish` calculation library (compiled to `dist/`)
- `web/` = the Next.js app (depends on the library via `file:..`)

The library must be built **before** the web app. That is already wired in
[`web/vercel.json`](web/vercel.json):

```json
{ "framework": "nextjs", "buildCommand": "npm run build --prefix .. && next build" }
```

No environment variables are required.

## Recommended: GitHub + Vercel dashboard

1. Create a new **empty** GitHub repo (no README/.gitignore).
2. Push this repo to it:
   ```bash
   git remote add origin https://github.com/<you>/<repo>.git
   git branch -M main
   git push -u origin main
   ```
3. On **vercel.com → Add New… → Project → Import** the repo.
4. In the import screen set **Root Directory = `web`** (click *Edit* → choose `web`).
   - Framework: **Next.js** (auto-detected).
   - Build/Install commands: leave default — Vercel reads `web/vercel.json` and
     installs the whole workspace automatically.
5. Click **Deploy**. You'll get a `https://<project>.vercel.app` URL.

Vercel auto-detects the npm workspace and includes the parent library in the build,
so the `npm run build --prefix ..` step finds it.

## Alternative: Vercel CLI

```bash
npx vercel login
npx vercel link          # choose/create the project
# set Root Directory to "web" when prompted (or in Project Settings later)
npx vercel --prod
```

## After deploy
Generate a chart and confirm the server-computed sections (KP, Jaimini, Transit,
Varshaphal) populate — these only appear when the chart is generated by the deployed
build.
